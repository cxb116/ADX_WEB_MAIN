## 3. 常见问题排查

### 问题1：定时任务完全没有执行

**可能原因：**
1. 数据库中没有配置定时任务
2. 应用没有重启
3. Bean名称配置错误
4. 定时任务被暂停（status=1）

**排查步骤：**

```sql
-- 检查定时任务配置
SELECT * FROM sys_job WHERE invoke_target = 'dataSspSlotCalculateTask.calculateCurrentMonth';

-- 检查定时任务状态（0=正常，1=暂停）
SELECT job_id, job_name, invoke_target, cron_expression, status, concurrent, create_time
FROM sys_job
WHERE invoke_target = 'dataSspSlotCalculateTask.calculateCurrentMonth';

-- 查看应用启动日志中是否扫描到了Bean
# 在应用启动日志中查找：DataSspSlotCalculateTask
```

### 问题2：定时任务执行但没有数据可计算

**可能原因：**
1. 当前月份的表不存在（如 data_ssp_slot_day_202604）
2. 表中没有数据
3. 没有满足条件的数据（分成模式、有展示PV、有千次收益、spend或income为空）

**排查步骤：**

```sql
-- 检查表是否存在
SHOW TABLES LIKE 'data_ssp_slot_day_202604%';

-- 检查表中是否有数据
SELECT COUNT(*) FROM data_ssp_slot_day_202604;

-- 检查是否有分成模式的数据
SELECT COUNT(*)
FROM data_ssp_slot_day_202604 t
INNER JOIN ssp_slot_info ssp ON t.ssp_slot_id = ssp.id
WHERE ssp.ssp_pay_type = 1;

-- 检查是否有需要计算的数据
SELECT COUNT(*)
FROM data_ssp_slot_day_202604 t
INNER JOIN ssp_slot_info ssp ON t.ssp_slot_id = ssp.id
WHERE ssp.ssp_pay_type = 1
  AND ssp.enable = 1
  AND t.show_pv > 0
  AND ssp.ssp_ecpm > 0
  AND (t.spend IS NULL OR t.spend = 0 OR t.income IS NULL OR t.income = 0);
```

### 问题3：定时任务执行但更新失败

**可能原因：**
1. SQL语法错误
2. 表名或字段名错误
3. 数据类型不匹配

**排查步骤：**
查看应用日志中的错误信息，通常会有详细的SQL错误。

### 问题4：Cron表达式不正确

**问题：**
- Cron表达式 `0 */1 * * * ?` 中的问号在某些Quartz版本可能不支持

**解决方案：**
使用明确的年月，例如：
- 2026年4月：`0 */1 * * * 202604`
- 或使用固定的cron表达式：`0 */1 * * * ?`（如果支持）

## 4. 手动测试定时任务

### 方法1：通过前端界面

1. 登录系统
2. 进入"系统管理" -> "定时任务"
3. 找到"媒体数据报表分成模式计算"任务
4. 点击"执行一次"按钮
5. 查看日志输出

### 方法2：通过SQL直接执行

查看定时任务日志表（如果存在）：

```sql
SELECT * FROM sys_job_log
WHERE job_id IN (SELECT job_id FROM sys_job WHERE invoke_target LIKE '%dataSspSlot%')
ORDER BY create_time DESC
LIMIT 10;
```

## 5. 日志级别配置

确保application.yml中的日志级别配置正确：

```yaml
# 日志配置
logging:
  level:
    com.ruoyi.system: debug  # 确保可以看到详细日志
    com.ruoyi.system.task: debug  # 定时任务日志
    com.ruoyi.system.service: debug  # Service层日志
    org.springframework: warn
```

## 7. 查看计算结果

定时任务执行后，可以查询计算结果：

```sql
SELECT
  t.id,
  t.ssp_slot_id,
  ssp.name AS slot_name,
  t.show_pv,
  ssp.ssp_ecpm,
  ssp.ssp_deal_ratio,
  t.spend,
  t.income
FROM data_ssp_slot_day_202604 t
INNER JOIN ssp_slot_info ssp ON t.ssp_slot_id = ssp.id
WHERE t.ssp_slot_id = 123
ORDER BY t.id DESC
LIMIT 10;
```

## 8. 总结

定时任务正常执行的标志：
1. 日志中每分钟看到"========== 定时任务开始执行 =========="
2. 日志中看到【Service层】相关的详细日志
3. 数据库中的spend和income字段被正确更新

如果定时任务没有执行，请按照上述步骤逐一排查。
